# Requirements Document

## Project Description (Input)
部活向け Discord Bot の新機能として、「/anna コマンドによる部内質問応答機能」を作りたいです。

### やりたいこと
ユーザーが Discord で /anna を実行し、質問文を入力すると、Bot が Azure 上の RAG を使って部内資料を検索し、部のマスコットキャラクター anna として自然な日本語で回答する機能を実装したいです。

### 背景
- 部員が部のルール、イベント情報、部内資料の内容、活動状況を手軽に確認できるようにしたい
- 情報が散らばっていて質問のたびに人に聞く運用を減らしたい
- 部のキャラクター性を活かした案内役として使いたい

### 想定ユーザー
- 部員
- 幹部
- 新入部員
- 将来的には管理者権限を持つメンバー

### 現在の前提
- Azure AI Foundry のプロジェクトは作成済み
- Azure AI Search も利用可能な状態
- リソースグループ内に関連リソースがある
- Discord Bot と Azure を接続するアプリケーションをこれから設計・実装する段階

### MVP の範囲
- Discord の /anna コマンドを受け付ける
- 質問文をバックエンドに送る
- Azure AI Search で関連文書を検索する
- 検索結果をコンテキストとして LLM に渡し、回答を生成する
- 回答は anna というキャラクターらしい口調で返す
- 回答に必要に応じて参照元タイトルを含める
- エラー時はユーザーにわかるメッセージを返す
- ログを残す

### MVP で扱わないもの
- 画像や音声の入力
- 複雑な管理画面
- 高度なアクセス制御
- 部員ごとの個人化
- 会話履歴に基づく長期記憶

### 成功条件
- 部員が /anna で自然に質問できる
- 部内資料を根拠にした回答が返る
- 資料にないことは「わからない」と言える
- キャラクターとしての一貫性がある
- Azure のコストを抑えた構成になっている
- 今後、資料追加や機能拡張がしやすい

### 特に設計で整理してほしいこと
1. Discord 側のインターフェース設計
2. バックエンド API の責務
3. Azure AI Search を使った RAG フロー
4. プロンプト設計とキャラクター制御
5. エラーハンドリング
6. ログ設計
7. 最小コストでの Azure 構成
8. 将来の拡張ポイント

### 要件定義で重視してほしい観点
- ユーザーストーリー
- 受け入れ条件
- 正常系 / 異常系
- セキュリティ
- コスト制約
- 運用しやすさ
- MVP と将来拡張の分離
